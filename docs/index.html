nuxt-auth-kit

Module Nuxt d'authentification clé-en-main pour API Laravel. Installez-le une fois — connexion, inscription, profil, mots de passe, rôles et permissions sont prêts à l'emploi.

Nuxt 3 & 4 Laravel Sanctum TypeScript Pinia RBAC

✦Fonctionnalités

Tout ce dont vous avez besoin, sans rien configurer manuellement.

🔐
Authentification complète
Login, register, logout avec persistance de session via cookie sécurisé.
👤
Gestion du profil
Mise à jour des infos, avatar et changement de mot de passe.
🔑
Reset de mot de passe
Flux complet : mot de passe oublié → email → réinitialisation.

⚙Configuration

Ajoutez le module et configurez l'URL de votre API Laravel.

nuxt.config.ts

ts
export default defineNuxtConfig({
  modules: [
    'nuxt-auth-kit',
    '@nuxt/ui'       // requis pour les styles Tailwind
  ],

  nuxtAuthKit: {
    // URL de base de votre API Laravel
    apiBase: process.env.NUXT_PUBLIC_API_BASE || 'http://localhost:8000',

    // Endpoints (valeurs par défaut, personnalisables)
    endpoints: {
      login:           '/api/auth/login',
      register:        '/api/auth/register',
      logout:          '/api/auth/logout',
      me:              '/api/auth/me',
      updateProfile:   '/api/auth/profile',
      updatePassword:  '/api/auth/password',
      forgotPassword:  '/api/auth/forgot-password',
      resetPassword:   '/api/auth/reset-password'
    },

    // Redirections automatiques
    redirects: {
      login:       '/auth/login',
      home:        '/',
      afterLogout: '/auth/login'
    },

    // Nom du cookie de session
    tokenCookieName: 'auth_token',

    // Contrôle d'accès par rôle
    rbac: {
      superAdminRole:  'super-admin',
      defaultUserRole: 'user'
    }
  }
})

.env

env
NUXT_PUBLIC_API_BASE=https://api.monprojet.com

◎useAuth()

Composable principal — auto-importé, aucun import nécessaire.

État réactif

Ref user
L'utilisateur connecté, ou null si déconnecté.
Ref<AuthUser | null>
Computed isAuthenticated
true si un utilisateur est connecté avec un token valide.
ComputedRef<boolean>
Computed isGuest
Inverse de isAuthenticated.
ComputedRef<boolean>
Ref loading
true pendant une requête en cours.
Ref<boolean>

Actions

async login(credentials)
Connexion. Redirige automatiquement vers redirects.home en cas de succès.
Promise<{ success: boolean, error?: ApiError }>
async register(data)
Inscription et connexion automatique. Redirige vers home.
Promise<{ success: boolean, error?: ApiError }>
async logout()
Déconnexion, suppression du cookie et redirection vers redirects.afterLogout.
Promise<void>
async fetchUser()
Récupère l'utilisateur depuis /api/auth/me. Utilisé au boot via le plugin.
Promise<AuthUser | null>
async updateProfile(data)
Met à jour le profil. Supporte File pour l'avatar (envoi en multipart/form-data).
Promise<{ success: boolean, error?: ApiError }>
async updatePassword(data)
Change le mot de passe. Requiert current_password, password, password_confirmation.
Promise<{ success: boolean, error?: ApiError }>
async forgotPassword(data)
Envoie l'email de réinitialisation. Requiert { email }.
Promise<{ success: boolean, message?: string, error?: ApiError }>
async resetPassword(data)
Réinitialise le mot de passe. Requiert { token, email, password, password_confirmation }.
Promise<{ success: boolean, message?: string, error?: ApiError }>

Exemple d'usage dans une page

vue
<script setup lang="ts">
// useAuth est auto-importé — pas besoin d'import
const { user, isAuthenticated, logout, loading } = useAuth()

async function handleLogout() {
  await logout()
  // redirige automatiquement vers /auth/login
}
</script>

<template>
  <div>
    <span v-if="isAuthenticated">Bonjour, {{ user?.name }}</span>
    <button @click="handleLogout" :disabled="loading">Déconnexion</button>
  </div>
</template>

◈Rôles & Permissions

hasRole() et hasPermission() sont accessibles partout via useAuth().

ts
const { hasRole, hasPermission } = useAuth()

// Vérifier un rôle unique
if (hasRole('admin')) { /* ... */ }

// Vérifier plusieurs rôles (au moins un)
if (hasRole(['admin', 'manager'])) { /* ... */ }

// Vérifier une permission
if (hasPermission('edit-posts')) { /* ... */ }

// Vérifier plusieurs permissions
if (hasPermission(['edit-posts', 'delete-posts'])) { /* ... */ }

Dans les templates

vue
<template>
  <!-- Visible uniquement pour les admins -->
  <AdminPanel v-if="hasRole('admin')" />

  <!-- Bouton conditionnel selon permission -->
  <button v-if="hasPermission('create-post')">
    Créer un article
  </button>
</template>

<script setup lang="ts">
const { hasRole, hasPermission } = useAuth()
</script>
✅

Le rôle super-admin (configurable via rbac.superAdminRole) bypasse toutes les vérifications de rôle dans le middleware role.

⟶Middlewares

Trois middlewares nommés disponibles dans definePageMeta.

auth
Redirige vers /auth/login si l'utilisateur n'est pas connecté. Conserve l'URL de retour dans ?redirect=.
guest
Redirige vers / si l'utilisateur est déjà connecté. À utiliser sur les pages login, register, etc.
role
Vérifie que l'utilisateur possède au moins un des rôles définis dans meta.roles.
ts
// Page accessible uniquement aux utilisateurs connectés
definePageMeta({ middleware: 'auth' })

// Page accessible uniquement aux visiteurs non connectés
definePageMeta({ middleware: 'guest' })

// Page réservée à certains rôles
definePageMeta({
  middleware: 'role',
  roles: ['admin', 'manager']  // au moins un des rôles requis
})

// Combinaison : connecté ET rôle requis
definePageMeta({
  middleware: ['auth', 'role'],
  roles: ['admin']
})

⬡API Laravel attendue

Endpoints par défaut, personnalisables via nuxtAuthKit.endpoints dans nuxt.config.ts.

MéthodeRouteAuth ?RéponseDescription
POST /api/auth/login — { user, token } Connexion
POST /api/auth/register — { user, token } Inscription
POST /api/auth/logout ✅ { message } Déconnexion
GET /api/auth/me ✅ { user } Utilisateur connecté
PUT /api/auth/profile ✅ { user } Mise à jour profil
PUT /api/auth/password ✅ { message } Changement mot de passe
POST /api/auth/forgot-password — { message } Email de réinitialisation
POST /api/auth/reset-password — { message } Réinitialisation avec token
ℹ️

Le champ user dans les réponses doit contenir au minimum id, name, email. Les champs roles et permissions sont optionnels (tableaux de strings) pour activer le RBAC.

⬡Exemple avec Laravel Sanctum

Implémentation de référence côté API Laravel.

routes/api.php

php
use App\Http\Controllers\AuthController;

Route::prefix('auth')->group(function () {
    // Routes publiques
    Route::post('login',          [AuthController::class, 'login']);
    Route::post('register',       [AuthController::class, 'register']);
    Route::post('forgot-password',[AuthController::class, 'forgotPassword']);
    Route::post('reset-password', [AuthController::class, 'resetPassword']);

    // Routes protégées (Sanctum)
    Route::middleware('auth:sanctum')->group(function () {
        Route::post('logout',  [AuthController::class, 'logout']);
        Route::get('me',      [AuthController::class, 'me']);
        Route::put('profile', [AuthController::class, 'updateProfile']);
        Route::put('password',[AuthController::class, 'updatePassword']);
    });
});

app/Http/Controllers/AuthController.php — méthode me()

php
public function me(Request $request): JsonResponse
{
    return response()->json([
        'user' => $request->user()->load('roles', 'permissions'),
    ]);
}

◇Types TypeScript

Tous les types sont exportés depuis nuxt-auth-kit.

ts
import type {
  AuthUser,
  LoginCredentials,
  RegisterData,
  UpdateProfileData,

  ResetPasswordData,

interface AuthUser {
  id:          number | string

↑Publication npm

Procédure complète pour publier et maintenir le package.

1
Installer les dépendances
npm install
2
Builder le package
Génère dist/module.mjs, dist/module.cjs et les types.
npm run build
3
Tester localement (recommandé)
# Option A — npm link
npm link                      # dans nuxt-auth-kit/
npm link nuxt-auth-kit        # dans votre projet

# Option B — yalc (plus fiable)
npm install -g yalc
yalc publish                  # dans nuxt-auth-kit/
yalc add nuxt-auth-kit        # dans votre projet
4
Se connecter à npm
npm login
npm whoami  # vérification
5
Publier
npm publish --access public
6
Mettre à jour (versioning)
npm version patch   # bug fix  : 1.0.2 → 1.0.3
npm version minor   # feature  : 1.0.2 → 1.1.0
npm version major   # breaking : 1.0.2 → 2.0.0
npm run build && npm publish
7
CI/CD avec GitHub Actions
Crée .github/workflows/publish.yml pour publier automatiquement sur chaque tag v*.
yaml
name: Publish to npm
on:
  push:
    tags: ['v*']
jobs:
  publish:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
      - uses: actions/setup-node@v4
        with:
          node-version: '20'
          registry-url: 'https://registry.npmjs.org'
      - run: npm ci
      - run: npm run build
      - run: npm publish --access public
        env:
          NODE_AUTH_TOKEN: ${{ secrets.NPM_TOKEN }}
ℹ️

Ajoutez NPM_TOKEN dans GitHub → Settings → Secrets → Actions. Générez-le sur npmjs.com → Account → Access Tokens → Automation.
